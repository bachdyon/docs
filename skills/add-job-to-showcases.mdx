## Mục đích

Skill này tạo một entry public trong [trang showcases](/showcases) từ một job đã hoàn thiện trong `jobs/<job_id>/`. Một showcase entry public chỉ giữ `output/`, đặc biệt là `output/final_video.mp4`, để tránh đưa nguyên liệu, transcript, metadata vận hành hoặc artifact nội bộ lên public.

Không copy `input/`, `source/`, `logs/`, `job.toml`, `remotion/`, `node_modules/`, hoặc file build/cache vào `showcases/`.

## Workflow

1. Xác định source `jobs/<job_id>/` và target `showcases/<job_id>/`.
2. Kiểm tra `output/final_video.mp4` tồn tại.
3. Copy riêng `jobs/<job_id>/output/` sang `showcases/<job_id>/output/`.
4. Verify target chỉ có `output/`.
5. Thêm player vào `docs/showcases.mdx` với URL raw từ repo `bachdyon/vas-showcases`.
6. Kiểm tra status ở root repo, `showcases/`, và `docs/`.

## Lệnh mẫu

```bash
mkdir -p showcases/<job_id>
rsync -a --delete jobs/<job_id>/output/ showcases/<job_id>/output/
```

```bash
find showcases/<job_id> -mindepth 1 -maxdepth 1 ! -name output -print
```

## Quality gates

- `showcases/<job_id>/output/final_video.mp4` tồn tại.
- `showcases/<job_id>/` chỉ có thư mục `output/`.
- Docs embed trỏ đúng `https://github.com/bachdyon/vas-showcases/raw/refs/heads/main/<job_id>/output/final_video.mp4`.
- Final response nói rõ `showcases/` và `docs/` là repo riêng nếu cần commit/push.
